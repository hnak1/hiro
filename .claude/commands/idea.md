# /idea

## 呼び出し方
`/idea "テーマ"` または `/idea "テーマ" --infinite`

## 概要
12の思考フレームワークを使ってアイデアを生成し、GitHub Issueに登録する。

## 12の思考フレームワーク

| # | フレームワーク | 問いかけ |
|---|--------------|---------|
| 1 | 深掘り | なぜそれが必要か？根本原因は？ |
| 2 | 応用 | 他の分野でこれを使えないか？ |
| 3 | 統合 | 2つのアイデアを組み合わせたら？ |
| 4 | 逆転 | 逆のアプローチをとったら？ |
| 5 | 極端 | 10倍・1/10のスケールにしたら？ |
| 6 | 他者視点 | ユーザー・競合・批評家はどう見る？ |
| 7 | ランダム接続 | 無関係な概念と結びつけたら？ |
| 8 | 制約追加 | 時間・予算・人数を半分にしたら？ |
| 9 | アナロジー | 自然界・歴史に似た事例は？ |
| 10 | 分解 | 要素に分解すると何が見える？ |
| 11 | 問題発見 | 誰も気づいていない問題は？ |
| 12 | 未来予測 | 5年後・10年後はどうなる？ |

## 処理手順

### 1. brain内関連ノート検索
- `knowledge-explorer` でテーマ関連情報を収集

### 2. フレームワーク選択・適用
- ランダムに3〜5個のフレームワークを選択
- 各フレームワークでアイデアを生成

### 3. --infiniteモード
- フレームワークを変えながら繰り返す
- ユーザーが停止するまで継続

### 4. 登録
- 重複チェック（既存Issueと照合）
- GitHub Issueに登録（または Markdown 出力）
- `00_Inbox/ideas-[テーマ]-[日時].md` に保存
