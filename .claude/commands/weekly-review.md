# /weekly-review

## 呼び出し方
`/weekly-review`

自動実行: 毎週金曜 18:00 JST（GitHub Actions）

## 概要
週次の振り返りを自動生成し、brain全体の健全性を確認する。

## 処理手順

### 1. Inbox確認
- `00_Inbox/` の未整理ファイル一覧を取得
- 滞留日数を計算

### 2. 活動集計
```bash
git log --since="7 days ago" --oneline
```
- コミット数・変更ファイル数・アクティブプロジェクト

### 3. プロジェクト進捗確認
- 各プロジェクトの進捗率を計算
- 7日以上更新のないプロジェクトを検出
- `/stale-check` を呼び出す

### 4. 週次レポート生成

```markdown
# 週次レビュー [日付]

## 今週のハイライト
-

## Inbox滞留
- 未整理: N件

## プロジェクト進捗
| プロジェクト | 進捗 | 最終更新 |
|------------|------|---------|

## 放置プロジェクト
-

## 来週の優先事項トップ3
1.
2.
3.
```

`10_Journal/weekly-[日付].md` に保存してgit commitする。
