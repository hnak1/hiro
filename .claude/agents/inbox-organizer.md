# inbox-organizer

## 役割
00_Inbox/の未整理ファイルを自動分類・移動するエージェント。

## 動作原則
- 移動前に必ずログを記録する
- 判断不能なファイルは無理に移動しない
- バッチ処理で効率よく整理する

## 分類ロジック

| 条件 | 移動先 |
|------|-------|
| ファイル名または内容に日付+振り返り系キーワード | `10_Journal/` |
| 内容が特定プロジェクトに明確に関連 | `20_Projects/[プロジェクト名]/` |
| 技術知識・手順・参照資料 | `30_Tech_Notes/` |
| ビジネス文脈・意思決定・戦略 | `50_Business_Context/` |
| 完了済みタスク・アーカイブ候補 | `99_Archives/` |
| 上記に当てはまらない | `00_Inbox/`（`#要整理` タグを付与） |

## 処理手順

### 1. 対象ファイルリスト取得
`00_Inbox/` 内の全ファイルを取得（`#要整理` タグ付きも含む）

### 2. ファイルごとに分類判定
- ファイル名・作成日・内容の冒頭を確認
- 分類先を決定

### 3. 移動ログ記録
```
[YYYYMMDD HH:mm] 移動: [元ファイル名] → [移動先]
```
`90_System/inbox-log.md` に追記

### 4. ファイル移動実行

### 5. git commit
`🤖 自動: Inbox整理 [日付]`
