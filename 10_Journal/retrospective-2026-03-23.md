## KPT

### Keep（続けること）

- スコアリング式による優先度の明示 → どのIssueを選ぶか判断が透明になる
- roadmapの `[~]` と `[x]` の区別 → ブレイン作業完了 vs ユーザー実行完了の明確化
- 各Issue完了後即座にroadmapを更新 → 状態の同期が保たれる

### Problem（問題点）

- **GitHub close権限制限**: `gh issue close` が全てのIssueで失敗する。roadmapとGitHubの状態が乖離し続けている
- **ユーザー依存タスク**: 上位Issueの多くが「数値記入・練習実施」などユーザー対応のもの。AIが単独で完了できるIssueが減ってきている
- **実質的に新規作業が少ない**: #12・#13は既に完了済みで確認のみ。#15のみ新規追加

### Try（次回試みること）

- GitHub close権限の問題 → ユーザーに手動クローズを依頼するチェックリストをroadmapに追記する
- 残課題の棚卸し → MS3（自律運用）の未着手Issue（brain健全性・自動リンク・意思決定ログ）にGitHub Issueを作成して次回実行対象に加える
- ユーザー実行待ちのIssueに「期限・リマインダー」を付ける仕組みを検討する
